# 从灵感到发布：我如何用 AI 重建写作流程

## 写作不应该这么难

你有没有过这样的经历：脑子里有个不错的想法，想写成文章，但打开编辑器就开始犹豫——从哪开始？要不要先做调研？配图怎么办？发到哪个平台？

结果就是，想法在草稿箱里躺了几个月，最后不了了之。

我也是这样，直到我发现可以把几个 Claude Code skills 组合起来，搭建一个从灵感记录到发布的完整流程。现在写作变成了："记下想法 → 回答几个问题 → 等待输出 → 点击发布"。

这篇文章就是用这个流程写出来的。

## 写作的四大摩擦点

在找到解决方案之前，我先经历了这些痛苦：

### 1. 启动困难

"要写什么？""怎么开头？""需要先做调研吗？"

每次想写东西，这些问题会在脑子里循环播放。结果就是打开编辑器，盯着空白页面发呆，最后关掉去刷微博。

拖延症不是懒，而是启动成本太高。

### 2. 准备工作耗时

真正开始写之前，需要：
- 收集相关资料
- 整理笔记和链接
- 搭建文章框架
- 想好论证逻辑

这些"准备工作"经常比写作本身还耗时。我试过一次花了两小时整理资料，结果因为太累当天没写一个字。

### 3. 发布焦虑

文章写完了，接下来呢？
- 微信公众号要配图、调格式
- X (Twitter) 有字数限制
- 小红书要做卡片
- 不同平台排版规则不一样

每次想到要在多个平台发布，就觉得"算了，下次再说"。然后就没有下次了。

### 4. 纯 AI 生成的空洞感

试过让 AI 直接写文章，但总觉得不对劲：
- 说的都对，但没有我的思考
- 很流畅，但缺乏真实感
- 看起来很专业，但读起来很空

AI 生成的内容可以骗过算法，骗不过读者。

**累积效应**：写作频率越来越低，从一周一篇变成一月一篇，再变成"有空再说"。

## 转机：技能生态的价值

转折点出现在我发现宝玉老师的几个 skills：

- **baoyu-xhs-images** - 自动生成小红书风格配图
- **baoyu-post-to-wechat** - 一键发布到微信公众号
- **baoyu-post-to-x** - 一键发布到 X

看到这些工具时，我意识到：如果能把我之前写的 **writing-assistant** 和这些发布工具串起来，就能打通整个写作流程。

不只是让某个环节更容易，而是让整个流程变得丝滑。

## 完整工作流：六个步骤

### 步骤 1：灵感捕捉

有想法时，随手在 markdown 文件里记下来。可以是：
- 一句话主题
- 几条零散笔记
- 一段粗糙的草稿

**关键原则**：降低记录门槛。不需要完美，只要别让想法溜走。

### 步骤 2：问答式写作

启动 `writing-assistant` skill，它会：

1. **分析你的内容** - 理解你想表达什么
2. **提出针对性问题** - 不是模板问题，而是基于你的内容
3. **收集你的回答** - 通过对话让思路清晰
4. **组织成框架** - 把零散想法结构化

这个过程像和编辑对话。你不需要知道"怎么写"，只需要回答"你想说什么"。

### 步骤 3：内容润色

调用 `content-research-writer` skill：
- 改善结构和流畅度
- 强化开头和结尾
- 增加必要的论证
- 提升整体质量

输出文件：`{filename}-polished.md`

**重要**：这不是重写，而是让你的表达更清晰。核心思想还是你的。

### 步骤 4：智能配图

使用 `baoyu-xhs-images` skill：
- 自动识别适合配图的位置
- 生成风格统一的视觉内容
- 避免过度配图影响阅读

手动配图可能要花半小时，现在几分钟搞定。

### 步骤 5：生成终稿

整合润色后的内容和图片：
- 图片插入到合适位置
- 确保排版合理
- 输出最终版本：`{filename}-final.md`

### 步骤 6：一键发布

选择发布平台：
- **微信公众号** → `baoyu-post-to-wechat`
- **X/Twitter** → `baoyu-post-to-x`

skills 会自动处理不同平台的格式要求。

## 实际效果：四个改变

使用这套流程一段时间后的真实变化：

### 1. 启动摩擦降低 90%

**以前**：想写 → 犹豫 → 拖延 → 放弃
**现在**：想写 → 记下来 → 回答问题 → 完成

关键是把"我要写一篇文章"变成"我回答几个问题"。后者容易太多。

### 2. 保留思考的价值

AI 不是替我写，而是帮我把想法表达得更清楚。

内容的核心——观点、经验、判断——都来自我自己。AI 只是让表达更流畅，让结构更合理。

这就是"AI 辅助"和"AI 生成"的区别。

### 3. 发布频率提高 3-4 倍

**以前**：一个月勉强写一篇
**现在**：一周可以写两三篇

不是因为我写得更快了，而是因为摩擦点都被消除了。

### 4. 专注内容而非格式

不用再花时间：
- 找配图素材
- 调整不同平台的排版
- 手动复制粘贴格式

可以把精力全部放在"我想说什么"上。

## 更深层的思考：AI 的正确使用方式

这套流程让我意识到一个重要原则：

**AI 应该是脚手架，不是替身。**

脚手架帮你更好地搭建建筑，但建筑的设计是你的。完工后脚手架会拆掉，留下的是你的作品。

如果让 AI 直接生成文章，那 AI 就成了替身。看起来也在"创作"，但那不是你的思考。

**这套流程的核心价值**，就是让 AI 帮你降低写作的摩擦，但保留思考的主导权。
